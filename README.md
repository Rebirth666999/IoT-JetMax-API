# 基于 Flask 的机械臂通信模块开发与架构解耦(IoT-JetMax-API)
智能工业物联网系统底层硬件接入模块。为 JetMax 吸盘机械臂开发的独立纯净后端，剥离了底层硬件控制与网络通信层，使其作为一个独立的微服务节点，无缝接入工业物联网大屏与业务系统。

### 1、**情景（Situation）：**

- 在“智能工业物联网系统”项目中，Jetmax 吸盘机械臂默认只能通过厂家闭源 APP 或底层命令行控制。为了将机械臂真正接入我们自己开发的业务流中，必须为其开发一套独立的控制后端。
- 当时实验室在硬件接入方面没有前人经验可以参考（从 0 到 1），且早期的人工智能辅助编程能力较弱，我完全依赖查阅官方散落的文档和网络开源社区进行摸索。
- **踩坑起点：** 我最初花了很多精力写了一个“前后端不分离”的方案——直接在本地跑一个带有简易前端按钮的 Web 页面来控制机械臂。但在代码评审（Code Review）时，负责整个系统架构的学长直接指出了致命问题：这种高耦合的页面根本无法接入系统，我们需要的是标准化的 API 接口。

### 2、**任务（Task）：**

- 废弃原有耦合架构，自学 Flask 框架，为 Jetmax 机械臂开发一套纯粹的 RESTful API 服务。
- 实现底层硬件动作与网络通信的彻底解耦，使机械臂能作为一个独立的“微服务”节点，接收整个物联网系统（如 Java/Vue 业务端）的网络指令并返回实时状态。

### 3、**行动（Action）：**

- 在明确了“解耦”的核心方向后，我通过 6 个版本的代码迭代，逐步完善了通信架构：
  - **V1版本：打破单体思维，建立通信规范**
    - **改进：** 抛弃 HTML 页面，引入 Flask 框架搭建后端路由（`/suck`、`/move_head` 等）。在这个阶段，我没有急于调硬件，而是先定义了前后端交互的 JSON 数据协议（包含 `name`、`type`、`require`），通过“接口 Mock”跑通了网络链路。
  - **V2-V3版本：引入面向对象与状态机，实现前后端状态同步**
    - **发现问题：** 在初步跑通网络控制后，负责系统架构的学长提出了进一步的业务需求：系统的前端大屏不能仅仅是“发送指令”，还必须能实时监控机械臂当前处于“空闲”、“工作中”还是“故障”状态，以便进行可视化展示。
    - **改进：** 为了响应这一需求，我引入了面向对象思想，将机械臂的核心逻辑封装成 `JetMax` 类，并在内存中开辟了专属的状态机（定义了 `STATUS_INIT`, `STATUS_WORKING`, `STATUS_IDLE` 等常量）以及坐标/抓取状态缓存。
    - **结果：** 这一改造让后端不仅具备了执行动作的能力，还能通过新增的 `/get_status` 等接口，随时向业务前端同步机械臂的物理生命周期。这彻底打通了从底层硬件到 Web 前端的状态闭环，完美支持了系统的动态 UI 展示。
  - **V4-V6版本：极致解耦，剥离通信层与控制层**
    - **改进：** 为了达到真正工业级的解耦标准，我将项目物理拆分为两个文件：
      1. **`JetMax.py`（控制层）：** 纯粹的硬件驱动类 `PArmSuck`，专门负责操作舵机、维护 `[x, y, z]` 坐标池和抓取状态（`capture`）。
      2. **`jetmax_control.py`（通信层）：** 专门负责解析 HTTP 请求，并通过实例化调用控制层的方法（如 `JetMax.test.MoveHead()`）来触发硬件。
    - **结果：** 代码结构变得极其清晰，哪怕未来更换 Web 框架或硬件主板，核心业务逻辑也无需推倒重来。

### 4、**结果（Result）：**

- 成功为机械臂开发出高可用的后端接口，与前端可视化大屏及 Java 业务端完美对接，实现了通过 Web 跨网段精准控制机械臂的抓取与坐标移动。
- **团队赋能：** 由于这套“分层解耦”的代码逻辑非常清晰，学长将其作为标准化模板推行给了实验室其他负责硬件开发的同学，帮助他们打破了迟迟无法完成硬件集成的瓶颈，大幅提升了整个团队的开发进度。

### 5、**复盘（Evaluation）：**

- **架构思维质变：** 这个经历是我从“为跑通功能堆砌代码（Scripting）”向“系统化工程设计（Software Engineering）”转变的转折点。我深刻体会到了高内聚、低耦合的威力，也正式在这个项目中确立了自己作为“解耦系统开发者”的角色定位。
- **自学与抗压能力：** 在缺乏成熟 AI 辅助和前人经验的“拓荒期”，面对学长的否定没有气馁，而是通过查阅原生文档和不断重构，硬啃下了这块硬骨头。这让我在此后的技术探索中，不再畏惧任何陌生的底层硬件或未知框架。

### 6、附加说明（addition）：

- 最终实现的效果是：通过postman测试接口，可以成功控制机械臂进行精确的抓取、释放、移动和复位。
